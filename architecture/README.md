# Mercury Architecture

This directory contains the architectural documentation for Mercury.

The purpose of this documentation is to explain the design of the platform, the reasoning behind key engineering decisions, and the principles that guide its evolution.

Rather than documenting implementation details, the architecture documentation focuses on answering **why** Mercury is designed the way it is.

---

## Documentation Structure

### Architecture Decision Records (ADRs)

Architecture Decision Records capture significant engineering decisions made during the development of Mercury.

Each ADR documents:

- the problem
- the available options
- the chosen solution
- the consequences of that decision

---

### Diagrams

Architecture diagrams provide visual representations of the platform, including:

- System architecture
- Data flow
- Data warehouse model
- Deployment architecture

---

### Engineering Principles

Engineering principles define the standards that guide the design and implementation of Mercury.

These principles ensure that architectural decisions remain consistent as the platform evolves.

---

## Guiding Philosophy

Mercury is intentionally designed as if it were the internal data platform of a growing e-commerce company.

Architecture is treated as a living part of the project.

As Mercury evolves, this documentation evolves with it.
